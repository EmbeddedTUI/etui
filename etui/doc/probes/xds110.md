# XDS110 Probe Guide

Use a Texas Instruments XDS110 probe or an XDS110-equipped LaunchPad to debug
supported MSPM0 targets through OpenOCD and LLDB.

## Supported Configurations

etui recognizes these XDS110 USB modes:

| Probe mode | OpenOCD interface |
|------------|-------------------|
| Native XDS110 | `interface/xds110.cfg` |
| CMSIS-DAP firmware | `interface/cmsis-dap.cfg` |

The Probe tab currently provides target presets for:

- MSPM0L
- MSPM0G
- MSPM0C

All three use SWD and the `thumbv6m-none-eabi` LLDB target triple.

## Prerequisites

1. Connect the XDS110 probe to the host and target.
2. Connect SWDIO, SWCLK, GND, and the appropriate target reference voltage.
3. Ensure the target is powered.
4. Open the **Tools** tab and verify that **OpenOCD** and **LLDB Debugger** are
   installed.
5. Install the operating-system USB/udev rules required for non-root probe
   access. etui cannot install system USB permissions.

If OpenOCD is installed outside `PATH`, add its `bin` directory under
**Settings → Tool Paths**, save, and run **Scan All** in the Tools tab.

## Start a Session

1. Open the **Probe** tab.
2. Click **Detect**.
3. Select the detected **TI XDS110** entry.
4. Confirm that the backend changes to **openocd**.
5. Select the target family: **MSPM0L**, **MSPM0G**, or **MSPM0C**.
6. Click **Start**.

etui launches OpenOCD with the selected interface, probe serial number, adapter
speed, ports, and MSPM0 target configuration. When OpenOCD reports that its GDB
server is ready, etui opens the **LLDB** tab and connects automatically.

## Default Ports

| Service | Default port |
|---------|--------------|
| GDB server | `3333` |
| OpenOCD Telnet | `4444` |
| OpenOCD TCL | `6666` |

Change these under **Settings → Probe / Debugger** when another process already
uses a default port.

## Load Firmware Symbols

Remote attachment works without a firmware image, but LLDB will not have source
paths, function names, or reliable instruction boundaries.

1. Build an ELF file containing debug information.
2. In the **LLDB** tab, enter its path in **Firmware ELF**.
3. Select **Load / Reconnect**.

Use a debug build with compiler options such as `-g` and avoid stripping the
ELF file.

## Stop the Session

Use **Stop** in the Probe tab before disconnecting the USB cable. This
terminates OpenOCD and releases the probe and configured ports.

## Troubleshooting

### XDS110 is not detected

- Confirm the USB cable supports data.
- Check that the probe appears in the host USB device list.
- Install or correct the host udev/USB permission rules.
- Disconnect other applications using the XDS110.
- Reconnect the probe, then click **Detect** again.

### Backend does not change to OpenOCD

Select the XDS110 entry rather than **Auto-detect**. Native XDS110 probes are
identified by USB VID:PID because pyOCD may not enumerate them.

### "Select a target MCU first"

Choose one of the MSPM0 target families before selecting **Start**. The target
selection determines the OpenOCD chip configuration and LLDB architecture.

### OpenOCD cannot find a configuration file

The selected OpenOCD installation must include its `scripts/interface` and
`scripts/target` data. A standalone executable without its scripts directory is
not sufficient.

### Port is already in use

Stop stale OpenOCD instances with **Kill stale**, or change the GDB, Telnet, and
TCL ports in Settings.

### LLDB aborts while reading remote memory

etui disables memory-backed dashboard panels, restarts the probe server, and
reconnects LLDB. Loading the firmware ELF is recommended because it supplies
symbols and code boundaries without relying entirely on remote-memory probing.
